## ⚠️ HIGH-RISK AREAS

### 1. AI System Prompts (HIGHEST RISK)
**Risk:** Mistranslation affects AI behavior significantly
- Prompts guide complex travel planning logic
- Schema field names must remain consistent
- Constraints (geospatial, time-based) are context-sensitive
- Example outputs are locale-specific ("蓉城三日·镜头里的成都")

**Mitigation:**
- Have expert review all prompt translations
- A/B test with sample inputs
- Document translation decisions
- Create version control for prompt changes

### 2. Form Validation & Error Messages
**Risk:** Longer translations break UI layout
- Some error messages could be 50%+ longer in English
- Form fields have space constraints

**Mitigation:**
- Design responsive form layouts
- Plan for text truncation/overflow
- Test all message variations

### 3. Date & Time Formatting
**Risk:** Locale-specific formatting inconsistencies
- "M月D日" (Chinese) vs "MM/DD" (US) vs "DD/MM" (EU)

**Mitigation:**
- Use date-fns locale support
- Centralize date formatting logic

### 4. Existing URLs & SEO
**Risk:** Adding [lang] segment breaks existing links
- Current: `/plans/123`
- New: `/en/plans/123` or `/zh-CN/plans/123`

**Mitigation:**
- Implement 301 redirects
- Set canonical tags
- Update sitemap.xml
