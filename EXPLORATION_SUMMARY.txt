🎯 核心数据流
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

用户交互流程 (5 步状态机)：

  form 
    ↓ 用户点击"生成行程"
  generating (流式 Prompt 预览 3-5 秒)
    ↓ 预览完成
  prompt-preview (用户可编辑确认)
    ↓ 用户点击"开始规划"
  planning (并行执行 5 个 Agent 30-45 秒)
    ├─ POI Agent (8-12 个地点推荐)
    ├─ Route Agent (每日行程规划)
    ├─ Tips Agent (打包清单 + 贴士)
    ├─ XHS Agent (小红书风格笔记)
    └─ Synthesis Agent (汇总编排)
    ↓ 所有 Agent 完成
  done (展示完整行程)

核心状态管理：
  • searchStore      → 表单参数 (localStorage 持久化)
  • itineraryStore   → 临时行程 JSON + activeDay
  • agentStore       → 5 个 Agent 的实时状态 + 流式输出
  • Supabase DB      → plans 表（完整行程 + 进度 + 参数）

🗄️ 数据库设计
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Supabase plans 表字段：
  • id            TEXT       plan-{时间戳}-{随机码}
  • device_id     TEXT       设备唯一标识（按设备隔离数据）
  • status        TEXT       pending | done | error | interrupted
  • title         TEXT       行程标题
  • destination   TEXT       目的地城市
  • start_date    TEXT       出发日期 (YYYY-MM-DD)
  • end_date      TEXT       返回日期 (YYYY-MM-DD)
  • days_count    INT        旅行天数
  • budget_*      NUMERIC    预算下限/上限
  • itinerary     JSONB      ✨ FullItinerary 完整 JSON
  • agent_progress JSONB     ✨ {poi: {...}, route: {...}, ...}
  • planning_params JSONB    用于恢复中断的规划
  • saved_at      TIMESTAMP  保存时间戳

关键 JSONB 结构：
  • FullItinerary: 最终行程对象（id, title, days[], xhsNotes[], budget）
  • agent_progress: 各 Agent 状态 {status, progress, message, preview}

🔗 API 端点总览
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

规划相关：
  GET  /api/plans?deviceId=xxx       查询我的计划列表
  POST /api/plans                     创建新计划（pending 或 done）
  GET  /api/plans/[id]               查询单个计划 + 实时进度
  PUT  /api/plans/[id]               更新计划状态
  DELETE /api/plans/[id]             删除计划

Agent 编排：
  POST /api/agents/preview-prompt    LLM 流式生成 Prompt 预览
  POST /api/agents/orchestrate       并行执行 5 个 Agent (SSE)
  POST /api/agents/orchestrate-bg    后台规划（可选）

单个 Agent（直接调用）：
  POST /api/agents/poi               POI 推荐
  POST /api/agents/route-plan        路线规划
  POST /api/agents/content           旅行贴士
  POST /api/agents/xhs               小红书笔记

外部服务：
  POST /api/amap/search              高德地图 POI 搜索
  GET  /api/amap/staticmap           高德地图静态地图

🤖 AI 引擎配置
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

默认：DeepSeek (成本低、速度快)
备用：Claude (质量高)

切换方式：.env.local 中设置 AI_PROVIDER=claude 即可

5 个 Agent 的 Prompt 模板：
  • POI Agent: destination + userPrompt → 推荐 8-12 个地点
  • Route Agent: POI list + dates → 生成每日行程
  • Tips Agent: destination + userPrompt → 打包清单 + 注意事项
  • XHS Agent: destination + prompt + days → 小红书风格笔记 (5 类型)
  • Synthesis: 4 个 Agent 输出 → FullItinerary JSON

📞 常见问题快速解答
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q: 项目的主要入口是哪里？
A: /src/app/page.tsx（主页）+ /src/hooks/useHomeFlow.ts（核心状态机）

Q: 如何修改 AI Prompt？
A: 编辑 /src/lib/agents/prompts/index.ts 中对应的 Agent prompt

Q: 如何添加新 Agent？
A: 参考 QUICK_REFERENCE.md 中的"添加新 Agent"步骤

Q: 数据如何持久化？
A: 搜索参数用 localStorage，完整计划用 Supabase plans 表

Q: 如何切换到 Claude？
A: .env.local 中设置 AI_PROVIDER=claude，并配置 ANTHROPIC_API_KEY

Q: 5 个 Agent 是如何并行执行的？
A: /api/agents/orchestrate 使用 Promise.all() 并行调用，通过 SSE 推送进度
